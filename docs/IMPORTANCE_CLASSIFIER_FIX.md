# Исправление проблемы "LLM вернул пустой ответ при классификации важности"


### Техническая причина

Некоторые модели, например модель **Yandex Cloud (deepseek-v4-flash)** возвращает пустой `content` (None) при запросах с очень маленьким параметром `max_tokens=10`. Это особенность работы данной модели - она не может генерировать такие короткие ответы.

### Где происходит

В файле `memory/manager.py`, метод `classify_importance()` (строки 142-180):
- Используется для автоматической оценки важности информации по шкале 1-10
- Вызывается при добавлении фактов в память и сохранении диалогов
- Использует `max_tokens=10` для получения короткого ответа (только число)

## Решение

### Вариант 1: Увеличить max_tokens (Рекомендуется)

Изменить параметр `max_tokens` с 10 до 50-100 токенов:

```python
# В файле memory/manager.py, строка 166
response = await self.llm_client.chat.completions.create(
    model=self.classifier_model,
    messages=[{"role": "user", "content": prompt}],
    temperature=0.1,
    max_tokens=50  # Было 10, стало 50
)
```

**Плюсы:**
- Модель сможет генерировать ответ
- Минимальные изменения кода
- Работает со всеми моделями

**Минусы:**
- Немного больше токенов используется (но это незначительно)

### Вариант 2: Использовать отдельную модель для классификации

Добавить в `.env` отдельную модель для классификации:

```env
# Используем более быструю модель для классификации
IMPORTANCE_CLASSIFIER_MODEL=gpt://b1gpesajkrdn6fsfnhqq/yandexgpt-lite/latest
```

**Плюсы:**
- Можно использовать более дешевую/быструю модель
- Гибкость в настройке

**Минусы:**
- Требует дополнительной настройки

### Вариант 3: Отключить автоматическую классификацию

Использовать фиксированную важность по умолчанию:

```python
# При добавлении фактов указывать важность явно
await memory.add_fact(
    content="Важная информация",
    category="general",
    importance=7  # Явно указываем важность
)
```

## Текущее поведение

При пустом ответе от LLM:
- ✅ Агент **продолжает работать** нормально
- ✅ Используется значение по умолчанию: `importance=5` (средняя важность)
- ⚠️ Выводится предупреждение в консоль
- ❌ Автоматическая классификация не работает
