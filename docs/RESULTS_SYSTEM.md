# 💾 Система сохранения результатов

## Проблема

Раньше агент не сохранял промежуточные результаты выполнения задач. Когда пользователь просил:
1. "Найди контакты транспортных компаний" → агент находил
2. "Сохрани это в файл" → агент **начинал искать заново**, так как не сохранил результаты первого поиска

Это приводило к:
- ❌ Повторному выполнению одной и той же работы
- ❌ Потере времени и ресурсов
- ❌ Невозможности использовать результаты между запросами

## Решение

Теперь агент имеет **систему сохранения результатов**, которая:
- ✅ Автоматически сохраняет результаты поиска и извлечения данных
- ✅ Позволяет использовать результаты в последующих запросах
- ✅ Показывает доступные результаты в контексте агента
- ✅ Автоматически очищает устаревшие результаты (TTL)
- ✅ Связывает результаты с задачами для лучшей организации

## Как это работает

### 1. Автоматическое сохранение результатов

Агент **автоматически** сохраняет результаты после:
- `browser_search_google` → сохраняет результаты поиска
- `browser_extract_content` → сохраняет извлечённый контент
- Формирования списков (компании, контакты и т.д.)

### 2. Использование сохранённых результатов

Когда пользователь просит сохранить данные в файл или использовать их иначе, агент:
1. Проверяет список сохранённых результатов (`result_list`)
2. Получает нужный результат (`result_get_latest` или `result_get`)
3. Использует данные без повторного поиска

### 3. Контекст результатов

Агент видит доступные результаты в своём контексте:
```
## Доступные результаты из текущей сессии:

1. **Список транспортных компаний** (тип: companies_list, создан: 2026-05-16 20:30:15)
   ID результата: 5
   Содержит 10 элементов
   Пример: {'name': 'ТК Деловые Линии', 'phone': '+7 (800) 250-00-00', ...}

💡 Используй result_get для получения полного содержимого результата.
```

## Инструменты для работы с результатами

### result_save
Сохраняет результат для последующего использования.

```python
result_save(
    result_type="companies_list",  # Тип результата
    content={"companies": [...]},   # Содержимое (dict, list или str)
    title="Список транспортных компаний",  # Заголовок
    task_id=5,  # ID связанной задачи (опционально)
    ttl_hours=24  # Время жизни в часах
)
# Возвращает: {"success": True, "result_id": 10}
```

**Типы результатов:**
- `search_results` - результаты поиска
- `extracted_data` - извлечённые данные со страниц
- `companies_list` - список компаний
- `contacts` - контактная информация
- `database_info` - информация о базах данных
- `file_content` - содержимое файлов
- `analysis` - результаты анализа
- Любой другой тип по необходимости

### result_get
Получает сохранённый результат по ID.

```python
result_get(result_id=10)
# Возвращает: {
#   "success": True,
#   "result": {
#     "id": 10,
#     "type": "companies_list",
#     "title": "Список транспортных компаний",
#     "content": {...},
#     "created_at": "2026-05-16 20:30:15"
#   }
# }
```

### result_list
Показывает список сохранённых результатов.

```python
result_list(result_type="companies_list", limit=10)
# Возвращает список результатов с краткой информацией
```

### result_get_latest
Получает последний сохранённый результат.

```python
result_get_latest(result_type="companies_list")
# Возвращает последний результат указанного типа
```

### result_delete
Удаляет сохранённый результат.

```python
result_delete(result_id=10)
# Возвращает: {"success": True, "message": "Результат #10 удалён"}
```

## Интеграция с задачами

Результаты можно связывать с задачами:

```python
# Создаём задачу
task_id = task_create(description="Поиск транспортных компаний")

# Сохраняем результаты с привязкой к задаче
result_save(
    result_type="companies_list",
    content=companies,
    task_id=task_id  # Связываем с задачей
)

# Позже можно получить все результаты задачи
results = get_task_results(task_id)
```

## Время жизни результатов (TTL)

По умолчанию результаты хранятся **24 часа**, после чего автоматически удаляются.

Можно изменить TTL при сохранении:

```python
# Сохранить на 1 час
result_save(..., ttl_hours=1)

# Сохранить на неделю
result_save(..., ttl_hours=168)

# Сохранить навсегда (не рекомендуется)
result_save(..., ttl_hours=999999)
```

## Структура данных

### Таблица task_results

```sql
CREATE TABLE task_results (
    id INTEGER PRIMARY KEY,
    task_id INTEGER,              -- ID связанной задачи (опционально)
    session_id TEXT NOT NULL,     -- ID сессии
    result_type TEXT NOT NULL,    -- Тип результата
    title TEXT,                   -- Заголовок
    content TEXT NOT NULL,        -- Содержимое (JSON)
    metadata TEXT,                -- Дополнительные метаданные
    created_at TIMESTAMP,         -- Время создания
    expires_at TIMESTAMP          -- Время истечения
);
```

## Преимущества

### 🚀 Нет повторной работы
Агент не повторяет поиск и извлечение данных, если они уже сохранены.

### 💾 Эффективное использование ресурсов
Экономия времени, API запросов и вычислительных ресурсов.

### 🔄 Гибкость использования
Результаты можно использовать в разных контекстах и форматах.

### 📊 Прозрачность
Пользователь видит, какие результаты доступны и может запросить их.

### 🧹 Автоматическая очистка
Старые результаты удаляются автоматически, не засоряя базу данных.

## Правила для агента

Агент **ОБЯЗАН** следовать этим правилам:

1. **После browser_search_google** → сохрани результаты через `result_save`
2. **После browser_extract_content** → сохрани данные через `result_save`
3. **После формирования списков** → сохрани через `result_save`
4. **Когда пользователь просит сохранить в файл** → СНАЧАЛА проверь `result_list` или `result_get_latest`
5. **НЕ ПОВТОРЯЙ поиск** если данные уже есть в результатах!

## Интеграция с памятью

Система результатов работает вместе с другими системами:

- **Память (MemoryManager)** - долговременные факты и знания
- **Задачи (TaskTracker)** - контекст выполнения задач
- **Результаты (TaskResultsManager)** - промежуточные данные

Когда результат становится важным, его можно сохранить в долговременную память:

```python
# Получаем результат
result = result_get(result_id)

# Сохраняем важную информацию в память
memory_save_fact(
    content=f"Найдены транспортные компании: {result['content']}",
    category="solution"
)
```

## Troubleshooting

### Результаты не сохраняются
1. Проверьте, что база данных создана: `memory/agent_memory.db`
2. Проверьте права доступа к файлу базы данных
3. Проверьте логи на наличие ошибок

### Результаты не найдены
1. Проверьте, что не истёк TTL (по умолчанию 24 часа)
2. Используйте `result_list()` чтобы увидеть все доступные результаты
3. Проверьте правильность session_id

### Агент всё равно повторяет поиск
1. Убедитесь, что используете последнюю версию main.py
2. Проверьте, что в system prompt есть правила работы с результатами
3. Агент должен видеть контекст результатов в промпте
