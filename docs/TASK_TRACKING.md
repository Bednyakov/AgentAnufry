# 📋 Система отслеживания задач

## Проблема

Раньше, если агент не мог выполнить задачу по какой-то причине, его нельзя было попросить повторить, так как он не сохранял контекст этого взаимодействия. Каждая новая попытка начиналась с нуля.

## Решение

Теперь агент имеет **систему отслеживания задач**, которая:
- ✅ Сохраняет контекст сложных задач
- ✅ Фиксирует все попытки выполнения
- ✅ Запоминает причины неудач
- ✅ Сохраняет только важную информацию (не засоряет память)
- ✅ Позволяет продолжить с того места, где остановились

## Как это работает

### 1. Автоматическое отслеживание

Агент автоматически создаёт задачи для сложных операций и фиксирует попытки их выполнения.

### 2. Умная фильтрация

Система сохраняет только:
- Описание задачи
- Выполненные действия
- Результаты попыток
- Ошибки и их причины
- Ключевые выводы

**НЕ сохраняется:**
- Весь диалог целиком
- Промежуточные технические детали
- Повторяющаяся информация

### 3. Повторные попытки

Когда вы просите агента повторить задачу, он:
1. Находит незавершённую задачу
2. Загружает контекст (что уже пробовали)
3. Анализирует прошлые ошибки
4. Пробует другой подход

## Примеры использования

### Пример 1: Неудачная установка пакета

```
💬 Вы: Установи пакет xyz через pip

🤖 Агент: [Создаёт задачу #1]
         [Пытается: pip install xyz]
         [Ошибка: пакет не найден]
         [Сохраняет попытку с ошибкой]
         Не удалось установить пакет xyz. Пакет не найден в PyPI.

💬 Вы: Попробуй ещё раз

🤖 Агент: [Загружает контекст задачи #1]
         [Видит прошлую ошибку]
         [Пробует альтернативу: поиск в GitHub]
         Нашёл пакет на GitHub, устанавливаю оттуда...
```

### Пример 2: Сложная многошаговая задача

```
💬 Вы: Настрой MongoDB и создай базу данных для проекта

🤖 Агент: [Создаёт задачу #2: "Настройка MongoDB"]
         [Попытка 1: установка MongoDB]
         [Успех: MongoDB установлен]
         [Попытка 2: запуск сервиса]
         [Ошибка: порт 27017 занят]
         [Сохраняет вывод: "Порт занят процессом X"]
         
         MongoDB установлен, но не могу запустить - порт занят.

💬 Вы: Освободи порт и продолжи

🤖 Агент: [Загружает контекст задачи #2]
         [Видит: порт занят процессом X]
         [Останавливает процесс X]
         [Запускает MongoDB]
         [Создаёт базу данных]
         [Обновляет статус: completed]
         [Сохраняет вывод: "Решение - остановить процесс X"]
         
         Готово! MongoDB запущен, база данных создана.
```

### Пример 3: Поиск похожих решений

```
💬 Вы: Как исправить ошибку "connection refused" в Docker?

🤖 Агент: [Ищет похожие задачи]
         [Находит задачу #15: "Исправление Docker connection refused"]
         [Загружает успешное решение]
         
         Я уже решал похожую проблему! Тогда помогло:
         1. Перезапуск Docker daemon
         2. Проверка firewall правил
         3. Использование host network mode
         
         Попробуем эти шаги?
```

## Инструменты для работы с задачами

### task_create
Создаёт новую задачу для отслеживания.

```python
# Агент использует автоматически для сложных задач
task_create(description="Установка и настройка PostgreSQL")
# Возвращает: {"task_id": 1}
```

### task_add_attempt
Фиксирует попытку выполнения.

```python
task_add_attempt(
    task_id=1,
    actions_taken=["apt install postgresql", "systemctl start postgresql"],
    result="Сервис запущен успешно",
    success=True
)
```

### task_add_insight
Сохраняет важный вывод.

```python
task_add_insight(
    task_id=1,
    content="Для PostgreSQL нужно сначала инициализировать кластер",
    insight_type="learning"
)
```

### task_update_status
Обновляет статус задачи.

```python
task_update_status(task_id=1, status="completed")
# Статусы: in_progress, completed, failed, blocked
```

### task_get_context
Получает контекст задачи.

```python
task_get_context(task_id=1)
# Возвращает: описание, попытки, ошибки, выводы
```

### task_list_incomplete
Показывает незавершённые задачи.

```python
task_list_incomplete()
# Возвращает список задач со статусом in_progress/failed/blocked
```

### task_search_similar
Ищет похожие успешные задачи.

```python
task_search_similar(description="установка базы данных")
# Возвращает похожие задачи со статусом completed
```

## Структура данных

### Таблица tasks
- `id` - уникальный ID задачи
- `session_id` - ID сессии
- `description` - описание задачи
- `status` - статус (in_progress/completed/failed/blocked)
- `created_at` - время создания
- `updated_at` - время последнего обновления
- `completed_at` - время завершения

### Таблица task_attempts
- `task_id` - ID задачи
- `attempt_number` - номер попытки
- `actions_taken` - список выполненных действий
- `result` - результат попытки
- `error_message` - сообщение об ошибке
- `success` - успешна ли попытка
- `context_snapshot` - снимок контекста

### Таблица task_insights
- `task_id` - ID задачи
- `insight_type` - тип вывода (learning/solution/blocker/workaround)
- `content` - содержание вывода
- `importance` - важность (1-10)

## Преимущества

### 🎯 Не теряется контекст
Агент помнит, что уже пробовал, и не повторяет те же ошибки.

### 💾 Экономия памяти
Сохраняется только важная информация, а не весь диалог.

### 🔄 Возможность повтора
Можно попросить агента продолжить задачу в любой момент.

### 📚 База знаний
Успешные решения сохраняются и используются для похожих задач.

### 🐛 Отладка
Легко понять, на каком этапе возникла проблема.

## Автоматическая очистка

Старые завершённые задачи (>90 дней) автоматически удаляются, чтобы не засорять базу данных.

Важные выводы из задач сохраняются в долговременную память через `memory_save_fact`.

## Где хранятся данные

Все данные хранятся в той же базе данных: `memory/agent_memory.db`

Можно просмотреть через любой SQLite клиент:
```bash
sqlite3 memory/agent_memory.db
.tables  # Покажет: tasks, task_attempts, task_insights
```

## Интеграция с памятью

Система задач работает вместе с долговременной памятью:
- **Память** - хранит факты, предпочтения, общие знания
- **Задачи** - хранят контекст конкретных операций

Когда задача завершается успешно, важные выводы автоматически сохраняются в память.

## Примеры команд

```bash
# Показать незавершённые задачи
💬 Вы: Покажи незавершённые задачи

# Продолжить задачу

# Найти похожие решения
💬 Вы: Как мы раньше решали проблемы с Docker?

# Получить детали задачи
💬 Вы: Что было в задаче #3?
```

## Технические детали

### Фильтрация информации

Система использует умную фильтрацию:
1. Сохраняет только последние 3 попытки (для краткости)
2. Обрезает длинные сообщения об ошибках (до 200 символов)
3. Хранит только топ-5 выводов по важности
4. Удаляет дубликаты действий

### Поиск похожих задач

Поиск работает по ключевым словам из описания задачи. Для более точного семантического поиска можно интегрировать с embeddings из MemoryManager.
