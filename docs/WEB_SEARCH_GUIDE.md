# Руководство по работе агента с веб-поиском

## Проблема

Агент не справлялся с задачами поиска в сети. Например, при задаче "найти 5 компаний грузоперевозчиков" агент:
1. ✅ Открывал браузер
2. ✅ Вводил правильный поисковый запрос
3. ❌ Находясь на странице с результатами поиска, не обрабатывал их
4. ❌ Отвечал, что не может найти нужную информацию

## Причина

Агент получал список результатов поиска (заголовки, ссылки, краткие описания) через `browser_search_google`, но **не извлекал полный контент** со страниц результатов. Он останавливался на кратких snippet'ах, которых недостаточно для получения детальной информации.

## Решение

### 1. Добавлен новый инструмент `browser_extract_content`

**Файл:** `tools/browser.py`

```python
async def browser_extract_content(url: str, selectors: list = None) -> Dict[str, Any]:
    """
    Извлекает текстовый контент со страницы по URL.
    Полезно для извлечения информации из результатов поиска.
    """
```

**Возможности:**
- Открывает страницу по URL
- Извлекает основной текстовый контент (до 3000 символов)
- Использует умные селекторы для поиска основного контента (article, main, .content и т.д.)
- Возвращает заголовок страницы и извлеченный текст

### 2. Интегрирован в систему агента

**Файл:** `main.py`

Добавлено:
- Импорт функции `browser_extract_content`
- Описание инструмента в массиве `TOOLS` с явным указанием "ОБЯЗАТЕЛЬНО используй после browser_search_google"
- Обработчик в функции `execute_tool`

### 3. Добавлены детальные правила в System Prompt

**Критически важный раздел:**

```
Правила работы с поиском в интернете (КРИТИЧЕСКИ ВАЖНО!):
1. Для поиска информации ВСЕГДА используй browser_search_google (НЕ browser_navigate!)
2. После получения результатов поиска ОБЯЗАТЕЛЬНО используй browser_extract_content для каждого релевантного URL
3. Алгоритм работы с поиском:
   Шаг 1: browser_search_google("твой запрос") → получаешь список результатов
   Шаг 2: browser_extract_content(url) для 3-5 релевантных ссылок → получаешь полный контент
   Шаг 3: Анализируй контент и формируй ответ
4. ПРИМЕР правильной работы:
   Задача: "Найди 5 компаний грузоперевозчиков"
   - browser_search_google("компании грузоперевозчики Россия")
   - browser_extract_content(url1), browser_extract_content(url2), browser_extract_content(url3)...
   - Извлекаешь названия компаний, контакты, услуги
   - Формируешь список из 5 компаний с деталями
5. НЕ ОСТАНАВЛИВАЙСЯ на результатах browser_search_google!
6. Используй browser_extract_content для ПОЛНОЙ информации
```

## Преимущества решения

### ✅ Полная информация
Агент получает не только заголовки и краткие описания, но и полный контент страниц с контактами, услугами, ценами и т.д.

### ✅ Явные инструкции
System prompt содержит пошаговый алгоритм с конкретным примером, что исключает неправильное использование инструментов.

### ✅ Автоматическое извлечение контента
Функция `browser_extract_content` умно определяет основной контент страницы, игнорируя навигацию, рекламу и футеры.

### ✅ Ограничение размера
Контент ограничен 3000 символами, что предотвращает переполнение контекста LLM.

### ✅ Обработка ошибок
Если страница не загружается или контент не извлекается, агент получает информативное сообщение об ошибке.

## Использование

### Для пользователя

Просто задавайте вопросы, требующие поиска в интернете:

```
💬 Вы: Найди 5 компаний грузоперевозчиков
💬 Вы: Какие есть Python фреймворки для веб-разработки?
💬 Вы: Найди информацию о последних новостях в AI
```

Агент автоматически:
1. Выполнит поиск в Google
2. Извлечет контент с релевантных страниц
3. Проанализирует информацию
4. Предоставит структурированный ответ

### Для разработчика

**Программное использование:**

```python
from tools.browser import browser_search_google, browser_extract_content

# Шаг 1: Поиск
results = await browser_search_google("ваш запрос")

# Шаг 2: Извлечение контента
for result in results["results"][:5]:
    content = await browser_extract_content(result["link"])
    print(f"Контент с {result['title']}:")
    print(content["content"])
```

**Кастомные селекторы:**

```python
# Извлечь контент только из определенных элементов
content = await browser_extract_content(
    url="https://example.com",
    selectors=[".article-body", "#main-content"]
)
```

## Возможные проблемы и решения

### Проблема: Агент все еще не извлекает контент

**Решение:** Проверьте, что:
1. Функция `browser_extract_content` импортирована в `main.py`
2. Обработчик добавлен в `execute_tool`
3. System prompt содержит правила работы с поиском

### Проблема: Контент не извлекается с некоторых сайтов

**Решение:** Некоторые сайты блокируют автоматизацию. Попробуйте:
1. Использовать другие результаты поиска
2. Добавить кастомные селекторы для конкретного сайта
3. Увеличить задержки в `browser.py`

### Проблема: Слишком много вызовов browser_extract_content

**Решение:** Агент должен извлекать контент только с 3-5 наиболее релевантных страниц. Если он извлекает больше, уточните в system prompt ограничение.

## Дальнейшие улучшения

Добавить поддержку DuckDuckGo, Bing, Yandex для случаев, когда Google блокирует запросы.
